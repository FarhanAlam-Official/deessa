## 1. Getting Started

### 1.1 Accessing the Story Editor

1. Log in to the admin panel at `https://deessa.org/admin`
2. Navigate to **Stories** in the sidebar
3. Click **Add Story** to create a new story, or click **Edit** on an existing story

### 1.2 Editor Interface Overview

The story editor consists of:

- **Toolbar**: Formatting and insertion controls at the top
- **Content Area**: Main editing space where you write your story
- **Metadata Fields**: Title, excerpt, category, image, etc.
- **Status Indicators**: Autosave status, character/word count
- **Action Buttons**: Preview, Save Draft, Publish

### 1.3 Creating Your First Story

1. Enter a **Title** for your story
2. Write a brief **Excerpt** (summary)
3. Select a **Category**
4. Upload a header **Image** (optional)
5. Start writing in the content editor
6. Use the toolbar to format your text
7. Click **Save Draft** or **Publish** when ready

## 2. Text Formatting

### 2.1 Inline Formatting

Apply formatting to selected text using toolbar buttons or keyboard shortcuts:

| Format | Toolbar Button | Keyboard Shortcut | Example |
|--------|---------------|-------------------|---------|
| **Bold** | **B** button | Ctrl+B (Cmd+B on Mac) | **bold text** |
| *Italic* | *I* button | Ctrl+I (Cmd+I on Mac) | *italic text* |
| <u>Underline</u> | U button | Ctrl+U (Cmd+U on Mac) | underlined text |
| ~~Strikethrough~~ | S button | None | ~~crossed out~~ |

**How to apply:**
1. Select the text you want to format
2. Click the toolbar button or press the keyboard shortcut
3. The formatting is applied immediately

**To remove formatting:**
- Select the formatted text and click the same button again

### 2.2 Headings

Use headings to structure your content:

| Heading | Toolbar Button | Usage |
|---------|---------------|-------|
| Heading 1 | H1 | Main title (use sparingly) |
| Heading 2 | H2 | Major sections |
| Heading 3 | H3 | Subsections |
| Heading 4 | H4 | Minor subsections |

**How to create a heading:**
1. Place cursor on a new line
2. Click the heading button (H1, H2, H3, or H4)
3. Type your heading text
4. Press Enter to return to normal paragraph

**Quick tip:** Type "/" to open the slash command menu and select a heading type.

### 2.3 Lists

Create organized lists for better readability:

**Bullet List:**
- Item 1
- Item 2
- Item 3

**Numbered List:**
1. First item
2. Second item
3. Third item

**How to create a list:**
1. Click the bullet list or numbered list button in the toolbar
2. Type your first item
3. Press Enter to create a new list item
4. Press Enter twice to exit the list

**Quick tip:** Type "- " (dash space) for bullet list or "1. " for numbered list.

### 2.4 Blockquotes

Use blockquotes for quotations or emphasized text:

> This is a blockquote. Use it for quotes, testimonials, or important callouts.

**How to create a blockquote:**
1. Click the blockquote button in the toolbar
2. Type your quoted text
3. Press Enter twice to exit the blockquote

### 2.5 Horizontal Dividers

Add visual separators between sections:

---

**How to insert a divider:**
1. Place cursor where you want the divider
2. Click the horizontal rule button in the toolbar
3. The divider is inserted automatically

## 3. Links

### 3.1 Adding Links

**To add a link to selected text:**
1. Select the text you want to link
2. Click the link button in the toolbar (🔗)
3. Enter the URL in the popup
4. Click "Insert" or press Enter

**To add a link without selecting text:**
1. Click the link button
2. Enter both the URL and link text
3. Click "Insert"

### 3.2 Link Types

**External links** (to other websites):
- Use full URL: `https://example.com`
- Automatically gets security attributes (rel="noopener noreferrer")

**Internal links** (to pages on your site):
- Use relative path: `/about` or `/stories/my-story`

**Email links:**
- Use mailto: `mailto:info@deessa.org`

### 3.3 Editing Links

**To edit an existing link:**
1. Click on the linked text
2. Click the edit icon in the bubble toolbar
3. Update the URL
4. Click "Update"

**To remove a link:**
1. Click on the linked text
2. Click the unlink button in the bubble toolbar

## 4. Images

### 4.1 Uploading Images

**To upload an image:**
1. Click the image button in the toolbar (🖼️)
2. Choose "Upload" tab
3. Click "Choose File" or drag and drop an image
4. Wait for upload to complete (progress bar shown)
5. Image is inserted automatically

**Image requirements:**
- Maximum file size: 5MB
- Supported formats: JPEG, PNG, WebP, GIF
- Recommended dimensions: 1200x800px or larger

### 4.2 Image from URL

**To insert an image from URL:**
1. Click the image button in the toolbar
2. Choose "URL" tab
3. Paste the image URL
4. Click "Insert"

### 4.3 Image Settings

After inserting an image, you can configure:

**Alt Text** (required for accessibility):
- Describes the image for screen readers
- Click on image → enter alt text in the field

**Caption** (optional):
- Displayed below the image
- Click on image → enter caption in the field

**Alignment:**
- Left: Image floats to the left
- Center: Image centered in content
- Right: Image floats to the right

**Width:**
- Small: 33% width
- Medium: 66% width
- Full: 100% width

**How to adjust image settings:**
1. Click on the inserted image
2. Use the controls that appear
3. Changes are applied immediately

## 5. Layout Blocks

Layout blocks help create visually diverse and engaging stories.

### 5.1 Two-Column Layout

Create side-by-side content sections:

**How to insert:**
1. Click the two-column button in the toolbar
2. Two empty columns appear
3. Click in each column to add content
4. Each column can contain text, images, lists, etc.

**Use cases:**
- Compare/contrast information
- Image alongside text
- Before/after scenarios
- Feature highlights

### 5.2 Callout Boxes

Highlight important information with styled boxes:

**Types:**
- **Info** (blue): General information or tips
- **Warning** (yellow): Cautions or important notes
- **Success** (green): Positive messages or achievements

**How to insert:**
1. Click the callout button in the toolbar
2. Select the type (info, warning, or success)
3. Type your content inside the callout
4. Press Enter twice to exit

**Use cases:**
- Important announcements
- Safety warnings
- Success stories
- Quick tips

### 5.3 Highlight Quotes

Create visually distinct quote blocks (different from standard blockquotes):

**How to insert:**
1. Click the highlight quote button in the toolbar
2. Type the quote text
3. Optionally add attribution (author name)

**Use cases:**
- Testimonials
- Key takeaways
- Impactful statements
- Expert opinions

### 5.4 Dividers

Insert styled horizontal dividers to separate sections:

**How to insert:**
1. Click the divider button in the toolbar
2. Divider is inserted at cursor position

**Use cases:**
- Section breaks
- Visual separation
- Content organization

## 6. Video Embeds

### 6.1 Embedding YouTube Videos

**How to embed a video:**
1. Click the video button in the toolbar (▶️)
2. Paste the YouTube URL
3. Click "Insert"
4. Video preview appears in the editor

**Supported URL formats:**
- `https://www.youtube.com/watch?v=VIDEO_ID`

### 6.2 Video Display

- Videos are embedded securely with sandbox attributes
- Visitors can play videos directly on the story page

## 7. Autosave & Draft Management

### 7.1 Autosave

The editor automatically saves your work:

- **Frequency**: Every 15 seconds after you stop typing
- **Status indicator**: Shows "Saving...", "Saved", or "Save failed"
- **Location**: Top right of the editor

**What is saved:**
- Story content
- Title, excerpt, category
- All metadata

**What triggers autosave:**
- Typing and then pausing for a moment
- Adding images or layout blocks
- Changing formatting

### 7.2 Local Storage Backup

For extra safety, content is also backed up in your browser:

- **Automatic backup**: Saved to browser local storage
- **Recovery**: If you close the browser accidentally, content can be recovered
- **Prompt**: You'll be asked if you want to restore unsaved content when you return

### 7.3 Unsaved Changes Warning

If you try to leave the page with unsaved changes:

- **Warning dialog**: Appears asking if you want to leave
- **Options**: Stay on page or leave (losing changes)
- **Disabled when**: All changes have been autosaved

## 8. Preview Mode

### 8.1 Using Preview

Preview shows exactly how your story will appear on the public site:

**How to preview:**
1. Click the **Preview** button
2. Story opens in a modal overlay
3. Review formatting, images, and layout
4. Click **Close** to return to editing

**What preview shows:**
- Same typography and styling as public page
- All formatting and layout blocks
- Images and videos
- Sanitized content (as visitors will see it)

### 8.2 Preview vs. Public View

**Preview shows:**
- Current unsaved changes
- Draft content (even if not published)

**Public view shows:**
- Only published stories
- Last saved version
- Sanitized and secure content

## 9. Publishing Workflow

### 9.1 Draft vs. Published

**Draft:**
- Not visible to public
- Can be edited freely
- Shows "Draft" badge in story list
- Displays created date

**Published:**
- Visible on public site
- Shows "Published" badge in story list
- Displays published date
- Can still be edited (updates immediately)

### 9.2 Publishing a Story

**To publish a new story:**
1. Complete all required fields (title, excerpt, category)
2. Review content in preview mode
3. Toggle **Published** switch to ON
4. Click **Save** or **Publish** button
5. Story is immediately visible on public site

**To unpublish a story:**
1. Open the published story
2. Toggle **Published** switch to OFF
3. Click **Save**
4. Story is hidden from public (but not deleted)

### 9.3 Publishing Checklist

Before publishing, verify:

- [ ] Title is clear and compelling
- [ ] Excerpt summarizes the story well
- [ ] Category is correct
- [ ] Header image is uploaded (if applicable)
- [ ] All images have alt text
- [ ] Links are working
- [ ] Content is proofread
- [ ] Preview looks correct
- [ ] Story is ready for public viewing

## 11. Tips & Best Practices

### 11.1 Content Structure

**Use headings hierarchically:**
- Start with H2 for main sections
- Use H3 for subsections
- Don't skip heading levels

**Break up long paragraphs:**
- Keep paragraphs 3-5 sentences
- Use lists for multiple items
- Add visual breaks with dividers

**Use layout blocks strategically:**
- Callouts for important information
- Two-column for comparisons
- Highlight quotes for testimonials

### 11.2 Images

**Optimize before uploading:**
- Compress images to reduce file size
- Use appropriate dimensions (1200x800px recommended)
- Choose descriptive file names

**Always add alt text:**
- Describe what's in the image
- Keep it concise (1-2 sentences)
- Important for accessibility and SEO

**Use captions when helpful:**
- Provide context or attribution
- Explain what the image shows
- Credit photographers if needed

### 11.3 Writing Tips

**Keep it scannable:**
- Use headings to break up content
- Use bullet points for lists
- Highlight key information with callouts

**Link strategically:**
- Link to relevant resources
- Use descriptive link text (not "click here")
- Check links before publishing

**Preview before publishing:**
- Always preview your story
- Check on different screen sizes if possible
- Verify all images and videos load

### 11.4 Performance

**For long stories:**
- Editor handles 5000+ words smoothly
- Autosave works in the background
- No need to manually save frequently

**Image uploads:**
- Wait for upload to complete before adding more
- Progress bar shows upload status
- Retry if upload fails
